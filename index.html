End-to-end
1
Admin authenticates t = 0s

Admin logs into the React dashboard using email/password via Supabase Auth. JWT session token is stored. Super admin sees all sites; regular admin only sees assigned sites enforced by RLS.

Supabase Auth JWT RLS enforced
2
Select target website t = 1s

Admin picks a website from the dropdown (e.g. "Tejados Madrid"). Dashboard loads all config, content blocks, SEO, and theme for that specific site_id.

Query: sites Filter: admin_site_access
3
Edit content, theme, SEO, layout t = 2–60s

Admin modifies fields via rich editors. Images upload to Supabase Storage and return a CDN URL. Color pickers update hex values. SEO fields update meta tags. Layout toggle switches template variants.

content_blocks site_config site_seo Storage upload
4
Click "Publish" — write to Supabase t = +0.1s

Changes are committed via upsert() to PostgreSQL. Supabase automatically broadcasts a postgres_changes event to all active listeners subscribed to this site_id channel. Audit log records the diff.

supabase.upsert() Realtime broadcast Audit log insert
5
Astro site receives event t = +1–5s

Each deployed Astro website has a React Island that maintains a persistent WebSocket subscription to its own site_id channel. On event received, the component fetches updated data and triggers re-render.

supabase.channel() .on('postgres_changes') React setState
6
UI updates — visitors see changes t = < 30s total

New content renders in DOM. CSS custom properties update theme colors instantly. SEO meta tags update in <head>. Zero downtime, no deployment required, no page reload for active visitors.

CSS Custom Properties Dynamic <head> Zero Downtime ✅

Realtime System

📡 Astro Site — Realtime Listener
// React Island inside Astro site // Each deployed site has this env variable set const SITE_ID = import.meta.env.PUBLIC_SITE_ID // e.g. "uuid-madrid-001" function useSiteContent() { const [content, setContent] = useState({}) const [theme, setTheme] = useState({}) useEffect(() => { // Initial load fetchSiteData(SITE_ID).then((data) => { setContent(data.content_blocks) setTheme(data.site_config) applyTheme(data.site_config) }) // Subscribe to realtime changes for THIS site only const channel = supabase .channel(`site-${SITE_ID}`) .on('postgres_changes', { event: '*', schema: 'public', table: 'content_blocks', filter: `site_id=eq.${SITE_ID}` }, (payload) => { setContent(prev => updateBlock(prev, payload.new)) }) .on('postgres_changes', { event: 'UPDATE', schema: 'public', table: 'site_config', filter: `site_id=eq.${SITE_ID}` }, (payload) => { applyTheme(payload.new) // instant color update! }) .subscribe() return () => channel.unsubscribe() }, []) return { content, theme } }
🎨 Dynamic Theme Application
// Called every time site_config changes from admin function applyTheme(config) { const root = document.documentElement.style // Colors root.setProperty('--color-primary', config.primary_color) root.setProperty('--color-secondary', config.secondary_color) // Fonts (inject Google Fonts link dynamically) loadGoogleFonts([config.font_heading, config.font_body]) root.setProperty('--font-heading', `'${config.font_heading}'`) root.setProperty('--font-body', `'${config.font_body}'`) } // In Astro template — CSS variables power everything /* globals.css */ h1, h2, h3 { font-family: var(--font-heading); } .btn-primary { background: var(--color-primary); } .text-brand { color: var(--color-primary); }
